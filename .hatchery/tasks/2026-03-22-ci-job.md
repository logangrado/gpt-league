## Objective

Set up a CI job so league members can propose config changes via PRs and get standings output without running anything locally. Results exposed as downloadable artifacts.

## Context

The `gpt-league` repo holds Jsonnet race configs and uses the external `gpt-racing` library to compute standings. The scoring tool requires four API secrets (`CLIENT_ID`, `CLIENT_SECRET`, `CLIENT_USERNAME`, `CLIENT_PASSWORD`).

## Summary

### Key decisions

- **Platform**: GitHub Actions (no extra infrastructure needed)
- **Trigger**: `pull_request` + `push` to `main`, path-filtered to `configs/**` and `pyproject.toml`
- **What gets scored**: only `.jsonnet` files under `configs/` that were added or modified in the push/PR (detected via `git diff --diff-filter=AM`); if no configs changed, scoring is skipped entirely
- **Artifact structure**: each config scores into `out/<config-name>/`, all bundled into one `standings` zip artifact (30-day retention)
- **PR comment**: posts (or updates) a comment with a direct link to the Actions run where the artifact can be downloaded; uses a hidden marker to avoid duplicate comments on reruns

### Gotchas

**SSH → HTTPS for gpt-racing**: `pyproject.toml` originally used `ssh://git@github.com/...`. Actions runners have no SSH keys. Changed to `https://` and regenerated `uv.lock`. Future dependencies should use HTTPS URLs.

**Explicit permissions drop defaults**: adding `permissions: pull-requests: write` to the job implicitly sets everything else to `none`, breaking `actions/checkout`. Must also explicitly list `contents: read`.
