## Objective

Set up a CI job so league members can propose config changes via PRs and get standings output without running anything locally. Results exposed as downloadable artifacts.

## Context

The `gpt-league` repo holds Jsonnet race configs and uses the external `gpt-racing` library to compute standings. The scoring tool requires four API secrets (`CLIENT_TOKEN`, `CLIENT_SECRET`, `CLIENT_USERNAME`, `CLIENT_PASSWORD`).

## Summary

### Key decisions

- **Platform**: GitHub Actions (no extra infrastructure needed)
- **Trigger**: `pull_request` + `push` to `main`, path-filtered to `configs/**` and `pyproject.toml` — avoids running on unrelated changes (README edits, etc.)
- **Artifact retention**: 30 days, enough for PR review cycles without accumulating indefinitely
- **Config scored**: `configs/gpt/26-1.jsonnet` (current season, matching existing `run.sh`)

### Gotcha: SSH → HTTPS for gpt-racing

`pyproject.toml` originally referenced gpt-racing via `ssh://git@github.com/...`. GitHub Actions runners have no SSH keys and cannot clone private/public repos over SSH by default. Changed to `https://github.com/...` and regenerated `uv.lock`. Future dependency additions should use HTTPS URLs.

### Files changed

| File | Change |
|------|--------|
| `pyproject.toml` | gpt-racing git URL: SSH → HTTPS |
| `uv.lock` | Regenerated to reflect new URL |
| `.github/workflows/scoring.yml` | New — Compute Standings workflow |
| `README.md` | Added CI usage, artifact download instructions, secrets setup table |

### Secrets required (repo admin must configure)

Go to **Settings → Secrets and variables → Actions** and add:
`CLIENT_TOKEN`, `CLIENT_SECRET`, `CLIENT_USERNAME`, `CLIENT_PASSWORD`

### Verification

Open a PR that changes any file under `configs/` → the **Compute Standings** check should appear. After it passes, download the **standings** artifact from the run summary to confirm HTML tables are present.
