# Task: fix-ci

**Status**: complete
**Branch**: hatchery/fix-ci
**Created**: 2026-04-27 11:43

## Objective

Fix the `update-season` CI job so that daily re-runs don't overwrite user edits
(e.g. `penalties:` blocks) made on the open PR.

## Context

The workflow checked out `main` and force-pushed to `auto/update-season` each run,
destroying any edits the user had added to the PR branch since the last run.

## Summary

**File changed:** `.github/workflows/update-season.yml`

**Three changes made:**

1. **`fetch-depth: 0`** on the checkout step — required so `git fetch origin auto/update-season`
   can resolve remote branch history.

2. **New step "Use existing PR branch if open"** — inserted between checkout and `setup-uv`.
   Uses `gh pr view` to detect an open PR for `auto/update-season`. If found, fetches and
   checks out that branch instead of staying on `main`. This means `update-season` runs
   against the already-edited PR branch, so races already present (including user-edited ones)
   are not re-added.

3. **"Open PR" step simplified** — replaced `git checkout -b "$branch"` with a conditional
   that only creates the branch when it doesn't already exist locally. Removed `--force` from
   `git push` so the push is a normal fast-forward, not a destructive overwrite.

**End-to-end behavior:**
- First run (no PR): starts from `main`, creates branch, pushes, opens PR — unchanged.
- Subsequent runs (PR open): switches to PR branch → only truly new races appended → pushes
  as a new commit, PR and all user edits preserved.
- After merge: branch deleted, next run starts fresh from `main`.

**Gotcha:** The `gh pr view` call needs `GH_TOKEN` set — the new step uses `secrets.ACTIONS_PAT`
(same token already used in the "Open PR" step).
