# Task: auto-pr

**Status**: complete
**Branch**: hatchery/auto-pr
**Created**: 2026-03-22 22:33

## Objective

Add a GitHub Actions workflow to run `gpt-scoring update-season` on Monday nights, detect config changes, and open a PR automatically.

## Context

The `gpt-scoring update-season` command finds new iRacing races for a season and appends them to a jsonnet config:

```
uv run gpt-scoring update-season ./configs/gpt/26-1.jsonnet 10301 128672
```

If a race is added, output includes lines like:
```
Added: subsession_id=84164646 (Road America)
```

## Summary

Created `.github/workflows/update-season.yml` with:

- **Schedule**: `0 4 * * 2` — Tuesday 4 AM UTC = Monday 10 PM CST / 11 PM CDT
- **Manual trigger**: `workflow_dispatch` for testing
- **SSH agent** (`webfactory/ssh-agent@v0.9.0`) wired to `GPT_RACING_DEPLOY_KEY` secret so `uv sync` can fetch the private `gpt-racing` dependency over SSH
- **uv** installed via `astral-sh/setup-uv@v5`
- **Change detection**: `git diff --quiet configs/gpt/26-1.jsonnet` gates the PR step
- **PR creation**: branch `auto/update-season-YYYYMMDD`, commit message = the `Added:` lines joined, PR title = same; uses built-in `GITHUB_TOKEN`

**Prerequisite (manual):** Add a read-only SSH deploy key for `logangrado/gpt-racing` as a repository secret named `GPT_RACING_DEPLOY_KEY`.

**Verification path:**
1. Merge and trigger via `workflow_dispatch`
2. Confirm SSH key works (uv install succeeds)
3. If no new races, workflow exits cleanly — no PR opened
4. To force-test the PR path: temporarily remove a race from the config so the command has something to add back
